# MDEB App

Self-hosted personal data manager. Outdoors/hiking + cartography + technology aesthetic.

## Commands

```bash
docker compose up --build
docker compose exec backend alembic upgrade head
```

## Alembic Migrations

```bash
docker compose exec backend alembic revision --autogenerate -m "description"
docker compose exec backend alembic upgrade head
```

**Gotchas:**
- Adding a value to an existing PostgreSQL enum cannot be autogenerated â€” write manually: `op.execute("ALTER TYPE <name> ADD VALUE IF NOT EXISTS '<value>'")`
- New models must be imported in `alembic/env.py` (`import app.<module>.models  # noqa: F401`) or Alembic won't detect them
- New routers must be registered in `app/main.py`

## Architecture

Backend is organized by module. Each module (`tasks`, `journal`, `finance`, `time_tracking`, `notes`, `health`, `timeline`) has:
- `models.py` â€” SQLAlchemy ORM models
- `schemas.py` â€” Pydantic request/response schemas
- `router.py` â€” FastAPI route handlers

Shared infrastructure lives in `app/core/`: `database.py` (Base, get_db, engine) and `config.py` (Settings).

Adding a new module:
1. Create `app/<module>/` with `__init__.py`, `models.py`, `schemas.py`, `router.py`
2. Import models in `alembic/env.py` for migration detection
3. Register router(s) in `app/main.py`

Other patterns:
- Upsert-by-date pattern: journal entries and day ratings use `PUT /api/<resource>/{date}`
- Finance transfers: link `account_id` â†’ `to_account_id`; category type = `transfer`
- Time entries: `date` = reporting date (user-assigned); `end_time` can be next calendar day
- Notes folders: self-referential `parent_id`; delete re-parents children to grandparent
- Timeline router aggregates across all modules â€” update it when adding new data models

## Design Theme

Custom Tailwind colors: `forest-*` (greens, sidebar), `earth-*` (ochre), `slate-*` (blues), `parchment-*` (aged paper, main bg)

CSS utility classes: `.topo-bg` (contour lines on sidebar), `.grid-bg` (coordinate grid on main content)

## Mobile

Target device: Pixel 6 (412px). Key breakpoint conventions:
- `md` (768px) â€” sidebar visibility, two-column layouts
- `sm` (640px) â€” modal sizing, secondary elements

Patterns:
- **Navigation**: sidebar is `hidden md:flex`; mobile bottom tab bar in `App.vue` (`md:hidden`). Primary tabs (4): Dashboard/Tasks/Journal/Notes. Secondary "More" drawer: Time/Finance/Timeline
- **Modals**: `items-end sm:items-center` + `rounded-t-xl sm:rounded-xl` for bottom-sheet on mobile
- **Data tables**: card-based `sm:hidden` alternative + `hidden sm:block` table
- **Main content**: `pb-24 md:pb-6` to clear the mobile bottom nav bar
